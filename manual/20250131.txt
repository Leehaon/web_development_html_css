CSS Part 시작

HTML 요소만으로 웹 페이지를 생성할 수 있지만, 기본 형태만 갖추어진 형태입니다.
시각적인 요소, 디자인적인 요소를 가지고 웹 페이지를 '꾸며주는' 실직적인 역할을 담당하는 것이 CSS. 

CSS를 적용하는 세 가지 방식

# 1 인라인 스타일(inline style)
  : HTML 문서의 태그가 있는 라인 마다 style 속성으로 CSS 코드를 일일이 넣어주는 배열
    여러 요소에 공통 속성을 부여할 수 없다는 단점.
    HTML과 CSS 코드가 분리되지 않기 때문에 제한적으로 사용.

# 2 내부 스타일 시트(internal style sheet)
  : HTML의 <head> 태그 내에 <style> 태그를 집어넣고, 그 안에 CSS 코드를 작성하는 방식.
    인라인 스타일의 단점을 보완하여 공통 속성을 부여할 CSS를 열광적으로 정리 가능.
    style이 너무 길어지게 될 경우에는 <head> 태그가 너무 길어져서 전체 HTML 문서의 열이
    증가 하기 때문에 style이 적은 경우에만 사용하는 편

# 3 링킹 스타일 시트(linking style sheet)
  : HTML 문서 파일과 CSS 파일을 각각 분리해서 작성한 후에 서로 연결(link)하여 사용.
    HTML과 CSS 코드를 분리할 수 있고,
    '한 번 작성한 CSS 코드를 여러 HTML 파일에서 콩통으로 사용 가능'하기 때문에 가장 널리 사용.

cssstyle.html
cssstyle.css

특히 # 3의 경우,
서로 분리된 두 파일을 연결하려면 <head> 태그 내에 <link> 태그를 넣어서(그래서 링킹스타일시트) 생성할 필요가 있습니다. 해당 경우에 rel="stylesheet"는 자동완성되는 편이고, href=""에 CSS 파일 경로를 작성해서 넣어주면 됩니다.

우리 예시 href="./cssstyle.css" 있습니다.

선택지 
  : CSS로 HTML 문서의 어떤 요소를 꾸미라고 할 때 꾸밀 대상을 올바르게 지정하기 위한 방식.
    'CSS가 특정 요소를 선택하여 스타일을 적용'하게 하는 중요한 역할.
  종류
    1. 기본 선택자(모두 다 '~자'임 내가 오타내서 '~지'라고 했을 수 있음) : 사실 기본선택자가 위에 있는 설명이긴합니다. seletor.html / seletor.css 파일 생성

    2. 태그 선택자 : HTML 문서에서 작성했던 p, span, em, a 등의 각각의 태그르 지정하여 style을 작성하는 방식

    3. 클래스 선택자 : HTML 문서 작성시에(springboot에서도 했습니다.) 태그 내에 class을 
    명시하는 경우가 있는데, id와 비슷해보이기는 합니다.
    하지만 클래스 선택는 페이즈이 여러 곳에 '중복으로 사용할 수 있음'
    클래스 명을 복수로 지정하고 싶다면 하나의 클래스명 작성 뒤 공백(스페이스바)을 한 칸 띄우고
    다음 클래스명 작성하시면 됩니다 -> 예시는 seletor.html에서 p blue dark 피드에 작성함.

    4. id 선택자 : 한 페이지에서 여러 요소를 공유해서 사용할 수 있는 클래스와 달리 id는 한 가지 요소에만 고유하게 사용. 샵(#)을 사용해서 지정합니다.

    5. 그룹 선택자 : 여러 요소를 한 꺼번에 선택하고 싶을 때
